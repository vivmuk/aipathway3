## Data Flow

### How Answers Are Processed:

1. **User Profile Generation** (`app.js` → `generateUserProfile()`)
   - Answers are mapped to a structured user profile object
   - Job description is stored as `jobDescription` field (can be `null`)

2. **Role Analysis** (`api.js` → `analyzeJobDescription()`)
   - If job description provided, AI analyzes it to extract:
     - Role summary
     - Key responsibilities
     - AI enhancement opportunities
     - Use cases
     - AI agents that can be built
     - Transformation timeline

3. **Course Personalization**
   - All chapters are personalized using:
     - Primary goal
     - AI experience level
     - Industry context
     - Specific challenge
     - Learning style
     - Technical background
     - Role analysis (if job description provided)

4. **Role-Specific Chapter**
   - If job description provided, an additional chapter (Chapter 11) is generated:
     - Title: "How AI Will Enhance Your Role: Use Cases and Agents You Can Build"
     - Content includes specific use cases and agent building guides for their role

## Question Types

### Single Choice (`type: 'single'`)
- Radio button selection
- Only one option can be selected
- Stored as: `string` (the selected option's `value`)

### Multiple Choice (`type: 'multiple'`)
- Checkbox selection
- Multiple options can be selected
- Stored as: `array` of selected option `value`s

### Text Area (`type: 'textarea'`)
- Free-form text input
- Used for job description
- Stored as: `string` (can be empty/null for optional questions)
